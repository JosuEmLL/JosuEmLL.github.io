Actividad 04

Evaluación de conocimientos sobre reglas de iptables

Lopez Lopez Josué Emmanuel

Introducción

Esta actividad tiene como objetivo evaluar los conocimientos adquiridos sobre las reglas de iptables, una herramienta fundamental en la gestión de seguridad de redes en sistemas Linux. Se espera que el estudiante comprenda cómo se definen y aplican las reglas de filtrado de paquetes para controlar el tráfico de red.

Desarrollo

Cronología del Ataque

  • Establecer una politíca de seguridad restrictiva

    • iptables -P INPUT DROP

    • iptables -P FORWARD DROP

    • iptables -P OUTPUT DROP

  • Permitir tráfico de conexiones ya establecidas

    • iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

  • Aceptar tráfico DNS (TCP) saliente de la red local

    • iptables -A OUTPUT -p tcp --dport 53 -j ACCEPT

  • Aceptar correo entrante proveniente de Internet

    • iptables -A INPUT -p tcp --dport 25 -j ACCEPT

  • Permitir correo saliente a Internet desde el servidor de correo

    • iptables -A OUTPUT -p tcp --dport 25 -j ACCEPT

  • Aceptar conexiones HTTPS desde Internet a nuestro servidor web

    • iptables -A INPUT -p tcp --dport 443 -j ACCEPT

  • Permitir tráfico HTTP desde la red local a Internet

    • iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT

Reflexiones del Alumno

A través de esta actividad, he podido comprender la importancia de las reglas de iptables en la gestión de la seguridad de redes. La capacidad de definir políticas de filtrado específicas me permite controlar el tráfico de red y proteger los sistemas contra posibles amenazas. Además, la práctica de aplicar estas reglas me ha ayudado a consolidar mis conocimientos sobre cómo funcionan las conexiones de red y cómo se pueden asegurar eficazmente.

Esta experiencia me ha permitido aplicar los conceptos teóricos aprendidos en el curso de seguridad de redes, especialmente en el manejo de reglas de filtrado y control de acceso a través de iptables. La comprensión del funcionamiento interno del sistema operativo Linux y su capacidad para gestionar tráfico de red mediante reglas específicas es fundamental para cualquier profesional en el área de seguridad informática.
