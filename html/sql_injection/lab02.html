Laboratorio 02

Evasión de login a través de SQLi

PortSwigger Web Security Academy

Introducción

Este laboratorio contiene una vulnerabilidad de inyección SQL en la función de inicio de sesión (login). La aplicación interactúa con la base de datos sin filtrar adecuadamente el nombre de usuario y la contraseña introducidos.

Desarrollo

Para resolver el laboratorio, debemos evadir la autenticación e iniciar sesión en la cuenta del usuario administrator.

Pasos, Pruebas y Payloads

1. Navegamos al apartado de inicio de sesión.

2. En el nombre de usuario, ingresamos el payload: administrator'--

3. En la contraseña, ingresamos cualquier valor, por ejemplo: 1234, y enviamos el formulario.

Explicación

La consulta de backend verifica las credenciales de la siguiente manera: SELECT * FROM users WHERE username = '[input_user]' AND password = '[input_pass]'. Al ingresar el payload, la consulta se vuelve: ... WHERE username = 'administrator'--' AND password = '...'. El doble guion (--) comenta la verificación de la contraseña, permitiendo el ingreso solo con el nombre de usuario.

Reflexiones del Alumno

En este laboratorio, aprendí a evadir la autenticación e iniciar sesión en la cuenta del usuario administrator mediante la inyección de payloads en parámetros de entrada.

Comprendí cómo el uso de caracteres especiales como el apóstrofe (') y el doble guion (--) puede alterar la lógica de la consulta SQL.
